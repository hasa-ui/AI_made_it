# AI_made_it ロードマップ

最終更新: 2026-03-24  
基準仕様書: `仕様書.md`  
対象バージョン起点: `Ver.1.26.0`

## 目的

現行版は `Prestige → Ascension → Celestial → Abyss` の骨格が揃っている。  
次の開発では、各層の役割を明確化し、終盤の選択肢と再挑戦性を増やす。

## 現状整理

強み:

- 多層構造とタブ UI が成立している
- Celestial 4ルートの土台がある
- Abyss と Abyss Challenge が存在する
- セーブ移行とデータ駆動の基盤がある

課題:

- Abyss gain が実質 1 固定
- Challenge 報酬が数値寄り
- Celestial のルート差が浅い
- Legacy Tree に排他ビルドがない
- 後半導線と推奨表示が不足

## マイルストーン

### Phase 1: 終盤進行の再整備

状態:

- 完了（2026-03-24）

目的:

- Abyss 到達後に遊びが続く状態を先に作る

実施項目:

- [完了] Abyss gain 式を再設計し、1固定状態を解消
- [完了] Abyss アップグレードを役割別に追加
- [完了] Abyss 到達後の短期目標を UI に表示
- [完了] Challenge 報酬のうち一部を機能解放型へ差し替え

完了条件:

- [達成] Abyss 到達後に 3 段階以上の次目標が提示される
- [達成] Abyss 実行回数を重ねる意味が明確になる

### Phase 2: Celestial ルートの本格分岐化

目的:

- 4ルートを「見た目の違い」ではなく攻略方針の違いにする

実施項目:

- Nova / Vault / Mirror / Epoch の専用アップグレードを拡充
- ルート切替時の有効効果を UI 上で明示
- ルート別の推奨プレイスタイルをヘルプと画面上に表示
- ルート別実績またはルート別到達目標を追加

完了条件:

- ルートごとに有利な進め方が説明なしでも分かる
- どのルートを選ぶかで中後半の攻略順が変わる

### Phase 3: Legacy / Challenge のビルド化

目的:

- 周回ごとの選択を増やし、単純な総倍率ゲームから脱却する

実施項目:

- Legacy Tree に排他ノードを追加
- Legacy Tree に Challenge 特化 / 放置特化 / 高速周回特化の枝を追加
- Challenge 報酬に保持・自動化・ルール変更系を追加
- Challenge 選択画面に狙いと報酬タイプを表示

完了条件:

- Legacy の買い方でビルドが分かれる
- Challenge が寄り道ではなく進行装置として機能する

### Phase 4: 導線と運用性の改善

目的:

- 後半の迷いを減らし、反復プレイの負荷を下げる

実施項目:

- 次に狙うべき行動の提案表示を追加
- Prestige / Ascend / Abyss の推奨タイミング表示を追加
- ビルドプリセット保存を追加
- 統計を拡張し、ルート別・Challenge別の実績を確認可能にする

完了条件:

- 中盤以降に「何をすればいいか分からない」状態が減る
- 周回準備の操作コストが減る

### Phase 5: 第5層 `Paradox` の試作

目的:

- 第5層を上位通貨ではなくルール改変層として成立させる

実施項目:

- `Paradox Fragment` の試作実装
- 法則スロットの仕様確定
- 3〜5 個の法則を試験実装
- セーブ / UI / 実績 / 統計の対応追加

完了条件:

- 第5層が既存層の焼き直しではない
- 周回ルールを変える価値がゲームプレイに出る

## 実装順の指針

1. 先に既存最終層の遊びを厚くする
2. 次に Celestial と Legacy をビルドゲームとして育てる
3. その後に導線と運用性を改善する
4. 最後に第5層を追加する

## 検証観点

各フェーズで最低限確認すること:

- セーブ互換が維持されること
- 既存進行が破綻しないこと
- 変更後の新しい目標が UI 上で見えること
- 追加要素が単純な倍率増加だけに終わっていないこと
