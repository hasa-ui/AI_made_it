# AI_made_it 仕様書

最終更新: 2026-03-24  
対象バージョン: `Ver.1.26.0`  
対象セーブスキーマ: `14`

## 1. 文書の目的

本書は、リポジトリ内の現行実装を基準にした仕様書である。  
過去の拡張構想を含むメモは整理し、以下の2つを分けて扱う。

- 現在ゲーム内で実装済みの仕様
- 次段階の拡張方針

ロードマップは本書を基準に別紙 `ロードマップ.md` で管理する。

## 2. ゲーム概要

`AI_made_it` は、Gold を稼ぎ、ユニットとアップグレードを購入し、`Prestige`、`Ascension`、`Celestial`、`Abyss` を進める多層型インクリメンタルゲームである。

現行版の基本方針は以下。

- 序盤は Gold と GPS の伸長を中心に進める
- 中盤は Prestige と Legacy Tree で周回速度を引き上げる
- 後半は Ascension と Celestial で恒久強化とビルド分岐を行う
- 終盤は Challenge と Abyss で制約突破と最上位成長を行う

現在はアルファ版であり、数値バランスと後半導線は継続調整中である。

## 4. コアループ

プレイヤーの基本進行は以下。

1. Gold を得る
2. ユニットを購入する
3. 通常アップグレードを購入する
4. Prestige で Legacy を獲得する
5. Legacy Tree と Resonance Layers で周回を高速化する
6. Ascension で AP と CP を獲得する
7. Celestial でルート選択と追加強化を行う
8. Challenge と Abyss で終盤進行を拡張する

### 5.1 通常進行

### 5.2 Prestige 系

### 5.3 Ascension 系

## 6. 通常進行仕様

### 6.1 初期値

- 初期 Gold: `50`
- オートセーブ間隔: `5000ms`
- オフライン進行上限: `24時間`

### 6.2 ユニット

現行ユニットは5種。

1. ジュニア採掘機
2. 採掘機
3. エクスカベーター
4. ドリルコア反応炉
5. ヴォイド掘削艦

各ユニットは以下の定義を持つ。

- `id`
- `name`
- `baseCost`
- `costMult`
- `baseGPS`
- `desc`

購入 UI は `+1`、`+10`、`Buy Max` に対応する。

### 6.3 通常アップグレード

現行の通常アップグレードは4種。

- 採掘効率向上
- 運営改善
- 反応炉オーバークロック
- ヴォイド超過給

効果種別は以下。

- `unitMult`
- `globalMult`

### 6.4 GPS 計算

現在の GPS は概ね以下を合成して算出する。

- ユニット基礎生産
- 通常アップグレード
- Legacy / Ascension / Celestial / Abyss / 実績 / Challenge 報酬
- Prestige 効果

Prestige 効果は別枠で乗算される。

`Prestige倍率 = 1 + (prestigeEarnedTotal × 1ポイントあたりのPrestige効果)`

初期の 1 ポイントあたり Prestige 効果は `0.05` で、各種恒久強化で上乗せされる。

## 7. Prestige 層

### 7.1 役割

Prestige は通常進行の最初のリセット層である。

- ユニットと通常アップグレードをリセットする
- Legacy を獲得する
- Resonance Layers の進行値を伸ばす
- 次周回の開始 Gold を恒久効果込みで再計算する

### 7.2 獲得式

総 Prestige 値は以下。

`floor(sqrt(totalGoldEarned / 2000))`

実際に今回得られる Legacy は以下。

`max(0, 総Prestige値 - 現在のprestigeEarnedTotal)`

### 7.3 Legacy Tree

現行の Legacy ノード数は `19`。  
主な効果カテゴリは以下。

- 全体倍率
- 特定ユニット倍率
- コスト軽減
- 開始 Gold 増加
- 固定 GPS 増加
- Prestige 効果加算

ノードは座標付きのツリーとして管理され、前提ノード条件を持つ。  
UI 上では SVG ツリーと Inspector で閲覧・購入する。

### 7.4 Legacy 上限拡張

Ascension Shop の `遺産限界突破理論` により、各 Legacy ノードの最大レベルを段階的に拡張できる。

### 7.5 Resonance Layers

現行の Resonance Layers は5段階。

- 残響層
- 織界層
- フラクタル層
- 軌道層
- 極光層

解放条件は `prestigeEarnedTotal` に基づく。  
効果種別は以下。

- global multiplier
- start gold
- prestige effect add
- cost multiplier
- flat GPS

Resonance Layers は `Ascension` と `Abyss` でリセットされる。

## 8. Ascension 層

### 8.1 役割

Ascension は Prestige より上位のリセット層であり、以下を担う。

- AP の獲得
- CP の獲得
- 恒久ブーストの購入
- 一部保持機能の解放
- 自動購入の解放
- 周回ログの記録

### 8.2 獲得式

Ascension gain の元式は以下。

`raw = sqrt(prestigeEarnedTotal / 25)`

その後、`20` 以降にソフトキャップを適用する。

- `raw <= 20`: `floor(raw)`
- `raw > 20`: `floor(20 + (raw - 20)^0.72)`

### 8.3 リセット内容

Ascend 実行時、基本的に以下をリセットする。

- Gold
- ユニット
- 通常アップグレード
- Prestige 値
- 所持 Legacy
- 累計 Gold
- Legacy Tree

ただし、以下は Ascension Shop 解放後に保持可能。

- 累計 Gold
- Legacy Tree

### 8.4 Ascension Shop

現行の Ascension アップグレード数は `17`。  
主なカテゴリは以下。

- 全体倍率
- 固定 GPS
- Prestige 効果加算
- 開始 Gold 増加
- 累計 Gold 保持
- Legacy Tree 保持
- Legacy 上限拡張
- 自動購入解放

### 8.5 自動購入

`自律運用OS` 購入で解放される。

さらに Abyss アップグレード `自律継承アーカイブ` を購入すると、Abyss reset 後も解放状態を維持できる。

対象:

- ユニット
- 通常アップグレード
- Legacy

設定項目:

- 有効 / 無効
- 対象選択
- 購入方式 `single` / `max`
- 実行間隔 (`50ms` 以上)

### 8.6 Ascension ミニゲーム

Ascension Shop 全購入後に解放される。

- 名称: `星脈同調`
- 開始コスト: `AP 1`
- ラウンド数: `10`
- レーン数: `4`
- 5ラウンドごとに反転ルールが出現
- スコアに応じて AP を最大 `6` 還元

記録する値:

- 挑戦回数
- ベストスコア
- 前回スコア
- 前回ミス数
- 完全勝利回数
- 最高連鎖

### 8.7 周回統計

Ascend 時に周回情報を履歴化する。

- 周回番号
- 到達 Gold
- 所要秒数
- 獲得 AP
- ノーアップグレード判定
- 使用ユニット種数
- Challenge 中 Ascend の有無
- 終了時刻

履歴は最新 `30件` を保持する。

## 9. Celestial システム

### 9.1 現在の位置づけ

Celestial は Ascension 配下のサブシステムである。  
Ascend 時に CP を獲得し、Celestial Layers と Celestial アップグレードで中後半ビルドを分岐させる。

### 9.2 CP 獲得式

`max(1, floor(ascGain / 3) + floor(unlockedCelestialLayers / 2))`

### 9.3 Celestial Layers

現行の Layers は4段階。

- ノヴァ層
- ヴォルト層
- ミラー層
- エポック層

解放条件は `ascEarnedTotal` に基づく。  
各層は恒久ボーナスを持つ。

### 9.4 ルート分岐

現行の Branch は4種。

- Nova
- Vault
- Mirror
- Epoch

ルートは対応 Layer の解放後に選択できる。  
現在の実装仕様は以下。

- ルート選択自体にコストはない
- 選択中ルートの固有ボーナスだけが有効になる
- ルート専用アップグレードは購入後も保存される
- ただし、ルート専用アップグレードの通常効果は選択中ルートに対応するものだけが有効
- 各ルートカードには推奨プレイスタイル、到達目標、保存済みだが待機中の専用強化を表示する

各ルートの役割:

- Nova: 高Tier火力と短い Ascend 周回
- Vault: コスト圧縮と量産安定化
- Mirror: 開始資源を厚くした再始動連打
- Epoch: Prestige 効率を積む長期ラン

### 9.5 Celestial アップグレード

現行数は `17`。

- 共通アップグレード
- Mirror 専用の開始 Gold 系
- Nova 専用の GPS / 最終 Tier 強化
- Vault 専用のコスト圧縮 / Excavator 強化
- Epoch 専用の Prestige 効率 / 全体倍率
- Mirror 専用の Ascension Shop 上限拡張
- Nova / Vault / Mirror / Epoch それぞれに追加の専用アップグレード 2種

補足:

- `星界チューニング規格` による Ascension Shop 上限拡張は、購入済みであれば他ルート選択時も有効
- 上限拡張の対象は繰り返し購入を前提とした Ascension Shop 項目のみで、一回きり (`maxLevel = 1`) の項目には適用しない
- Celestial アップグレード一覧では、各項目が「常時有効」「現在有効」「購入済み・待機中」のいずれかで表示される

## 10. Challenge

### 10.1 現行構成

現行の Challenge は `11種`。

- 通常 Challenge: `8種`
- Abyss Challenge: `3種`

### 10.2 共通ルール

Challenge 開始時は以下をリセットする。

- ユニット
- 通常アップグレード
- 所持 Legacy
- Prestige 値
- 累計 Gold

開始 Gold は通常の開始 Gold 計算を使う。  
ただし Challenge に `forceStartGold` がある場合はその値を優先する。

### 10.3 クリア・中断時の挙動

Challenge 開始前の `totalGoldEarned` は退避される。  
クリアまたは中断時は退避値を復元し、Challenge 状態だけを解除する。

現在の実装では、Challenge ラン中に購入したユニットやアップグレードはそのまま残る。

### 10.4 効果と報酬

制約の主な種類:

- 全体生産低下
- アップグレード禁止
- コスト上昇
- 単一路線固定
- 開始資金固定
- ユニット総数依存の減衰
- ユニット総数依存のコスト加速
- 最高 Tier のみ生産

報酬の主な種類:

- global multiplier
- flat GPS
- prestige effect add
- cost multiplier

注意:

- Challenge 報酬は通常の周回では恒久だが、`Abyss` 実行時には Challenge 完了状態ごとリセットされる

## 11. Abyss 層

### 11.1 解放条件

累計 Gold が `Number.MAX_VALUE` 到達時に解放される。  
表示上の目安は `1.8e308`。

### 11.2 役割

Abyss は現行の最上位リセット層である。

- Abyss Shard の獲得
- 深淵専用アップグレードの購入
- 後半ビルドの再構成

### 11.3 獲得量

現行の Abyss gain は、Abyss 到達時の基礎値 `1` に加えて以下を加算する。

- Challenge クリア数 4件ごと
- Celestial Layer 解放数 3件ごと
- Abyss Reset 回数 4回ごと
- Infinity 到達時の追加値
- Abyss gain 系アップグレード

Challenge / Celestial の節目は、Abyss reset で消える現在値ではなく、Abyss 間で保持する milestone を参照する。

### 11.4 リセット内容

Abyss 実行時、以下をリセットする。

- Gold
- ユニット
- 通常アップグレード
- Legacy / Legacy Tree
- 累計 Gold
- Prestige 値
- AP / Ascension Shop
- CP / Celestial / Celestial Branch
- Challenge 状態（進行中セッションと通常 Challenge のクリア情報）
- 現在周回状態

以下は維持される。

- 実績
- Abyss Shard
- Abyss アップグレード
- Abyss Challenge のクリア判定 / 最速秒

### 11.5 Abyss アップグレード

現行数は `13`。

カテゴリ:

- コスト圧縮
- 全体倍率
- 開始 Gold
- 固定 GPS
- Prestige 効果加算
- 自動化保持
- Excavator / Drillcore / Voidrig の個別強化

### 11.6 Abyss Challenge

現行の Abyss Challenge は3種。

- Dimensional Drain
- Tickspeed Debt
- Abyssal Singularity

Abyss reset 後も、Abyss Challenge のクリア判定と最速秒は保持される。

## 13. UI / UX / セーブ仕様

### 13.1 プレイ UI

プレイタブは「比較してから行動する」情報優先レイアウトを採用している。

- ユニットは所持数 / 生産寄与 / 次価格を表示
- 通常アップグレードは現在レベル / 次価格を表示
- Legacy は Inspector 付き SVG ツリー
- UI 更新は「常時軽い更新」と「表示中タブだけの重い再描画」に分離し、一覧系パネルは非表示中に再描画しない

### 13.2 モバイル対応

- Legacy Tree はズームイン / ズームアウト / リセットに対応
- モバイルでは transform ベースで拡大表示を行う

### 13.3 設定

設定画面で変更できるのは以下。

- 数字表示形式 `compact` / `scientific`
- 各確認ダイアログの個別 ON / OFF
- 実績トースト通知
- オフライン報酬通知
- 購入通知
- 一般トースト通知

### 13.4 セーブ

セーブ機能は以下を持つ。

- LocalStorage 保存
- 定期オートセーブ
- 短いデバウンス付きの操作保存
- JSON ダウンロード
- クリップボードコピー
- テキストインポート
- ファイルインポート
- 全データリセット

### 13.5 マイグレーション

セーブデータはバージョン管理される。

- 現行 `SAVE_VERSION = 16`
- 新しいセーブスキーマのデータはインポート拒否
- `Infinity` / `-Infinity` / `NaN` を保存可能
- 旧セーブからの不足キー補完を行う
- `Ver.1.26.0` 時点では旧 Celestial ルート専用アップグレードを CP 返金付きで再配置する移行を持つ
- `Ver.1.28.0` 時点では一回きり Ascension Shop 項目の過剰購入を AP 返金付きで補正する移行を持つ
- `Ver.1.29.0` では Celestial 分岐強化のみのため、セーブスキーマ番号は据え置き
- `Ver.1.29.1` では UI/保存の軽量化のみのため、セーブスキーマ番号は据え置き

### 13.6 アップデート表示

未確認バージョンがある場合、起動時に最新更新内容モーダルを表示する。

## 15. 現行課題

現行仕様として把握しておくべき課題は以下。

- Challenge 報酬は数値強化中心で、機能報酬がまだ少ない
- Legacy Tree は排他分岐やビルド固定化が未導入
- 終盤導線はヘルプ依存がまだ強く、推奨アクション提示が弱い

## 16. 今後の拡張方針

現行仕様を踏まえた次段階の方針は以下。

### 16.1 Abyss の拡張

- 深層アップグレードをさらに増やす
- Abyss 後の中期目標を追加する
- Challenge と Abyss の接続を強める

### 16.2 Challenge の再設計

- 数値報酬だけでなく機能報酬を導入する
- 高難度攻略が次層解放条件に結び付くよう再整理する

### 16.3 第5層の追加

将来的な第5層は、単なる上位通貨ではなくルール改変型のメタ層として設計する。  
仮称は `Paradox` とし、法則スロット方式を候補とする。

## 17. 最重要原則

今後の実装は以下を優先する。

1. 各層に明確に異なる役割を持たせる
2. 後半ほど「数字の増加」だけでなく「選択の変化」を増やす
3. セーブ互換を維持しながら拡張する
4. 仕様書は実装と同期し、将来案は別管理または明示分離する
